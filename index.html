GTD TV Studio — Privacy Policy

Effective date: 10 May 2026 · Last updated: 23 May 2026

English Español

This page covers GTD Launcher, GTD Plugin for Emby and GTD Setup (collectively, "the apps"), all developed and published by GTD TV Studio.

Short version: we do not collect, transmit or sell any personal data. Everything you configure stays on your device. The launcher does send anonymous crash reports and basic app-usage telemetry to Google's Firebase service so we can fix bugs and understand which features are used — never any content of your library, searches or credentials. Optional features that talk to third parties (Google Drive, Emby, TMDB, football APIs) only do so when you enable them, and you control the credentials.

1. Data we collect about you

The GTD Launcher uses Firebase Crashlytics and Firebase Analytics to receive anonymous crash diagnostics and basic usage telemetry. The GTD Plugin for Emby and the GTD Setup companion include neither and collect nothing.

What's transmitted from the launcher:

  • Crashlytics — when the app crashes: stack trace, native crash dump, device manufacturer + model, Android version, free memory at the time of the crash, app version, an anonymous installation UUID (regenerated when you uninstall). No account, no email, no library content, no search queries.
  • Analytics — anonymous app-instance ID, country (derived from IP, not stored), app open / session events, app version, OS version, device brand + model. Used to see things like "how many users updated to the latest version" or "is the new search screen actually used". No personal identifiers.

Both can be disabled at the OS level via Settings → Privacy → Usage & diagnostics on Android TV. There is no in-app toggle yet — we may add one.

4. Third-party services

The apps integrate with several third parties, each optional and triggered explicitly by you:

  • Google Play Billing — used to verify Premium purchases. Governed by Google's Privacy Policy.
  • Google Drive API — only invoked if you enable Cloud backup. Stores a single JSON file with your settings in your own Drive's app-folder. We never see it. Governed by Google's Privacy Policy.
  • Your Emby server — the plugin connects to whatever URL you provide. Communication goes directly between your device and your server.
  • The Movie Database (TMDB) — optional. Only called if you provide your own API key. TMDB privacy policy.
  • football-data.org / api-football.com — optional. Only called if you provide your own API key. Governed by their respective privacy policies.
  • Apple Aerials — the screensaver fetches publicly-available 4K nature videos hosted by Apple. No account, no identifier.
  • Firebase Crashlytics + Firebase Analytics (launcher only) — Google's crash-reporting and basic-usage-analytics services, used to fix bugs and understand which features are used. Data is anonymous (see §1 for the exact fields). Governed by Google's Firebase Privacy Policy.

5. Data we sell or share

Nothing for advertising or to humans. The crash + usage telemetry described in §1 is processed by Google's Firebase pipeline strictly for the purpose of improving the launcher. We never sell, share, license or hand-off data to any other third party for advertising or analytics.

6. Children

The apps are not directed at children under 13. They have no age-verification because they collect no personal data in the first place.

7. Your rights

The crash + usage telemetry (see §1) is keyed only to an anonymous installation UUID and an anonymous app-instance ID — both regenerate every time you uninstall and reinstall, so there's no persistent link to your identity. You can wipe everything by uninstalling the app; for the cloud-backup feature, delete the JSON file from your own Google Drive's app folder. We hold no other personal data about you, so there's nothing else to delete, export or correct on our side.

8. Changes to this policy

If we ever change how the apps handle data, we'll update this page and bump the "Last updated" date. Material changes will also be highlighted in the in-app changelog.
